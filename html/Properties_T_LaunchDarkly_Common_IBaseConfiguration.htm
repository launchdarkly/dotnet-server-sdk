IBaseConfiguration Properties

The IBaseConfiguration type exposes the following members.

Properties
  NameDescription
AllAttributesPrivate
Whether or not user attributes (other than the key) should be private (not sent to the LaunchDarkly server). If this is true, all of the user attributes will be private, not just the attributes specified with the AndPrivate... methods on the User object. By default, this is false.
BaseUri
The base URI of the LaunchDarkly server.
EventCapacity
The capacity of the events buffer.
EventFlushInterval
The time between flushes of the event buffer.
EventQueueCapacity Obsolete.
Deprecated name for EventCapacity.
EventQueueFrequency Obsolete.
Deprecated name for EventFlushInterval.
EventSamplingInterval Obsolete.
Deprecated. Enables event sampling if non-zero.
EventsUri
The base URL of the LaunchDarkly analytics event server.
HttpClientHandler
The object to be used for sending HTTP requests. This is exposed for testing purposes.
HttpClientTimeout
The connection timeout.
InlineUsersInEvents
True if full user details should be included in every analytics event. The default is false (events will only include the user key, except for one "index" event that provides the full details for the user).
Offline
Whether or not this client is offline. If true, no calls to Launchdarkly will be made.
PrivateAttributeNames
Marks a set of attribute names as private. Any users sent to LaunchDarkly with this configuration active will have attributes with these names removed, even if you did not use the AndPrivate... methods on the User object.
ReadTimeout
The timeout when reading data from the EventSource API. The default value is 5 minutes.
ReconnectTime
The reconnect base time for the streaming connection.The streaming connection uses an exponential backoff algorithm (with jitter) for reconnects, but will start the backoff with a value near the value specified here.
SdkKey
The SDK key for your LaunchDarkly environment.
StreamUri
The base URL of the LaunchDarkly streaming server.
UserKeysCapacity
The number of user keys that the event processor can remember at any one time, so that duplicate user details will not be sent in analytics events.
UserKeysFlushInterval
The interval at which the event processor will reset its set of known user keys. The default value is five minutes.
